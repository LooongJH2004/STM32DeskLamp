

# 📊 毕业设计数据处理全流程操作指南

本指南涵盖了从环境配置到生成论文插图的所有 Python 执行指令。

## 1. 环境初始化 (仅需执行一次)

在 VS Code 中打开 `Thesis_Data_Analysis` 文件夹，打开终端并执行：

### 1.1 创建虚拟环境
强制使用 Python 3.12 确保稳定性：
```bash
# Windows 系统
py -3.12 -m venv .venv

# Mac/Linux 系统
python3.12 -m venv .venv
```

### 1.2 激活虚拟环境
```bash
# Windows (PowerShell)
.\.venv\Scripts\activate

# Mac/Linux
source .venv/bin/activate
```
*激活后，终端提示符前应出现 `(.venv)` 字样。*

### 1.3 安装依赖库
使用你创建的 `requirements.txt` 安装所有工具：
```bash
pip install -r requirements.txt
```

---

## 2. 运行数据处理脚本 (核心指令)

请确保你的原始串口日志已分别存放在 `data/` 文件夹下的对应 `.txt` 文件中。

### 2.1 运行 7.1.1 PWM 线性度与恒功率测试
**输入要求**：手动填写的 `data/pwm_*.csv` 文件。
**执行指令**：
```bash
python plot_pwm_analysis.py
```
**产出**：`output/pwm_linearity.pdf` 和 `output/pwm_colortemp_algo.pdf`。

### 2.2 运行 7.1.2 CRC 误码拦截压力测试
**输入要求**：将包含 `[TX_ERROR]` 的日志存入 `data/serial_error_test.txt`。
**执行指令**：
```bash
python plot_7_1_2_crc_test.py
```
**产出**：`data/parsed_crc_errors.csv` (中间表) 和 `output/crc_interception_test.pdf` (论文插图)。

### 2.3 运行 7.1.3 编码器高频交互吞吐量测试
**输入要求**：将包含 `{"ev":"enc",...}` 的日志存入 `data/serial_encoder_test.txt`。
**执行指令**：
```bash
python plot_7_1_3_encoder_test.py
```
**产出**：`data/parsed_encoder_events.csv` (中间表) 和 `output/encoder_throughput_test.pdf` (论文插图)。

## 3. 常用维护指令

### 3.1 更新依赖
如果你修改了 `requirements.txt`，请重新运行：
```bash
pip install -r requirements.txt
```

### 3.2 退出虚拟环境
当你完成数据处理工作后：
```bash
deactivate
```

### 3.3 强制重新生成所有图表 (一键指令)
如果你想一次性运行所有脚本，可以在终端输入：
```bash
python plot_pwm_analysis.py; python plot_7_1_2_crc_test.py; python plot_7_1_3_encoder_test.py
```

---

## 4. 故障排查 (Troubleshooting)

*   **找不到模块 (ModuleNotFoundError)**：确保你已经运行了 `activate` 指令进入了虚拟环境。
*   **图片中文乱码**：脚本中已配置 `SimHei` (黑体)。如果报错，请确保电脑安装了中文字体，或将脚本中的 `font.sans-serif` 修改为 `Arial Unicode MS`。
*   **PDF 无法打开**：如果 PDF 正在被 LaTeX 编译器占用，Python 脚本会写入失败。请先关闭 PDF 查看器或停止 LaTeX 编译再运行脚本。

---

**补充**：生成的 `.pdf` 文件是矢量格式，直接插入 LaTeX 能够获得印刷级的清晰度，建议优先使用 PDF 格式。
